Buuctf usualCrypto

  • 一道Buuctf里面的题目，usualCrypto,查壳发现无壳32位，打开main函数

  • 逻辑很简单，就是输入一串字符，然后通过sub_401080()函数处理后与字符串byte_40E0E4进行比较，其中字符串byte_40E0E4为”zMXHz3TIgnxLxJhFAdtZn2fFk3lYCrtPC2l9”

  • 进入sub_401080()函数

  • 大致看一下是进行base64加密，但是加密之前和之后分别有一个函数，加密之前是sub_401000()函数

是对base64的标准字符表进行修改，这里BASE64_table_40E0A0变量其实不单单只是A–J十个字符，应该是A–Za–z1–9+/,

  • 加密之后是sub_401030()函数

对加密之后的字符进行大小写转换

  • 所以解密脚本如下
import base64
table = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/='
model = list("ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/=")
s = "zMXHz3TIgnxLxJhFAdtZn2fFk3lYCrtPC2l9".swapcase()
for i in range(6,15):
    model[i], model[10 + i ] = model[10 + i], model[i]
model = ''.join(model)
table = str.maketrans(model, table)
print (base64.b64decode(s.translate(table)))

Buuctf IgniteMe

  • 一道Buuctf里面的题目，IgniteMe,查壳发现无壳32位，寻找main函数，发现没有main函数所以打开start函数
  • 分析一下，主要逻辑就是你输入字符串，程序调用sub_4010F0()函数对输入的字符串进行处理，然后通过sub_401050()函数进行判断
  • 先进入sub_4010F0()函数

sub_4010F0()函数就是去除输入字符串中的换行符和回车符

  • 进入sub_401050()函数

也没有什么复杂的逻辑，就是把字符串从尾部开始异或，每一项都为当前项和其后一项异或的值，其中最后一项和V4进行异或

  • 这里唯一的困惑就是V4的值，V4的值是通过函数sub_401000()产生的，函数反汇编后就一行大概是移位的操作，我们可以用IDA直接调试出来

在sub_401000()中设置一个断点，然后单步执行最后得到寄存器EAX中的值为4，即V4的值为4

  • 所以解题脚本如下
byte_403000 = [0x0D,0x26,0x49, 0x45, 0x2A, 0x17, 0x78, 0x44, 0x2B, 0x6C, 0x5D,0x5E, 0x45, 0x12, 0x2F, 0x17, 0x2B, 0x44, 0x6F, 0x6E, 0x56, 0x9,0x5F, 0x45, 0x47, 0x73, 0x26, 0x0A, 0x0D, 0x13, 0x17, 0x48, 0x42,0x1, 0x40, 0x4D, 0x0C, 0x2, 0x69, 0x0]
V4 = 0x4
flag = ""
len1 = len(byte_403000) - 1
byte_403000[len1] = byte_403000[len1] ^ V4
flag += chr(byte_403000[len1])
for i in range( len1 ):
    byte_403000[len1 - i - 1] = byte_403000[len1-i] ^ byte_403000[len1 - i - 1]
    flag += chr(byte_403000[len1-i-1])
print(flag[::-1])

Buuctf Overlong1

  • 一道Buuctf里面的题目，Overlong1，查壳发现无壳，32位，所以我们放到ida里，依旧是没有main函数，所以我们打开start函数
  • 主要逻辑也很容易懂，就是对unk_402008这个变量进行sub_401160()函数处理然后输出，输出的结果是

  • 仔细观察发现这条语句后面有个冒号 “:” ,估计是后面还有其他东西没有输出来，而再回到start()函数，unk_402008这个变量长度是0xAF位,而程序只处理了0x1C位，所以应该就是让我们修改二进制代码将unk_402008这个变量全部处理一下即可

  • 先开始我直接用ida修改代码，发现出现错误

  • 我也不知道为什么。。。所以就用OD去修改试了下,将28即0x1C修改为0xAF,运行一下发现得到flag
  • 当然这道题其实你要是分析sub_401160()函数的话然后自己再对完整的unk_402008变量进行处理也能得到结果

Buuctf hamburger competition1

  • 一道Buuctf里面的题目，hamburger competition1,是一种之前从来没有见过的题型，网上搜索了解了是Unity3D类安卓游戏逆向，这种题目的做法参考了博客 简单Unity3D类安卓游戏逆向思路，本题就是用ILSpy软件打开附件BJD hamburger competition_Data\Managed目录下的Assembly-CSarp.dll文件
  • 因为对这个软件一点也不了解，做这种类型的的题目也没有经验，所以一时间不知道如何下手，我就在搜索里面尝试性的输入了BJDCTF，然后找到了关键代码所在的地方

  • 逻辑很清晰了，就是如果str字符串用Sha1()函数加密后为DD01903921EA24941C26A48F2CEC24E0BB0E8CC7，那么flag就是str字符串用md5()函数加密，其中点进去查看会发现Sha1函数就是Sha1加密，而md5()函数是md5加密然后取前二十个字符

  • 所以对DD01903921EA24941C26A48F2CEC24E0BB0E8CC7进行Sha1结果为1001，再对1001进行md5加密，结果有四种

  • 分别取前20个字符套上flag{}去提交，发现最终的答案为flag{B8C37E33DEFDE51CF91E}
